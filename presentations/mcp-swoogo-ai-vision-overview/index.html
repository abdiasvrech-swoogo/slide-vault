Solving real API problems

Fuzzy Search

3-stage escalation: exact → expanded → full scan. Confidence scoring.

Grouped Questions

Flat API response → grouped by page. ~40% fewer tokens.

Translation Fallback

Empty "" translations → base language value. Sees labels, not blank strings.

Response-Diff

Only show changed fields. Detect silently ignored fields/mismatches.

Normalized Maps

PHP [] vs {} distinction. Converts empty arrays to objects for keyed maps.

Expand Params

Hydrate relations inline: fees, speakers, location. Eliminate N+1 queries.

Improving Swoogo API for everyone

Technical choices that shaped the architecture

  • AI SDK v6 over Mastra

    Dec 2025 release closed the gap. We extracted patterns without importing complexity.

  • Server-driven HITL

    AI SDK's needsApproval requires React. Our clients are headless AI hosts (Claude Desktop).

  • One connection per account

    Provides natural isolation and zero cross-tenant leakage risk by design.

"Node.js chosen for SSE performance — 10-40x advantage over PHP for streaming"

Critical discoveries during development

Laravel Limits

InvokingTool events lacked cancellation mechanisms. Forced migration to Node.js for security.

Context Bloat

Monolithic documentation led AI to hallucinate. Solved with composable scoping.

Persistent Tokens

MCP hosts persist sessions. Epoch-based Redis strategy ensures logout actually invalidates sessions.

Each problem led to architecture improvement
